🔊 EchoJS JSX Runtime Examples

These examples demonstrate the fine-grained reactive JSX runtime. Each example is a standalone page that you can open directly.

Note: These examples use Bun's bundler to compile TSX on the fly. Make sure you have run bun install in the project root first.

How to Run

Option 1: Using Bun Dev Server (Recommended)

# From project root
cd packages/jsx-runtime
bun dev  # or: bun run --hot example/server.ts

Option 2: Build and Serve

# Build the library first
cd packages/jsx-runtime
bun run build

# Then serve examples with any static server
cd example
python -m http.server 8080
# or: npx serve .

Option 3: Using Vite

cd packages/jsx-runtime/example
npx vite --open

Architecture Overview

┌─────────────────────────────────────┐
│           Your TSX Code             │
│  const Counter = createComponent(   │
│    () => ({ count: signal(0) }),    │
│    (vm) => <button>{vm.count}</button>
│  );                                  │
└──────────────┬──────────────────────┘
               │
               ▼ JSX Transform
┌─────────────────────────────────────┐
│      @echojs/jsx-runtime            │
│  • jsx() - creates real DOM nodes    │
│  • effect() - fine-grained updates   │
│  • NO virtual DOM, NO diffing        │
└──────────────┬──────────────────────┘
               │
               ▼ Direct DOM
┌─────────────────────────────────────┐
│         Real DOM Updates            │
│  • Text nodes updated in place       │
│  • Attributes updated reactively     │
│  • Components have reactive scopes   │
└─────────────────────────────────────┘

Built with ❤️ using fine-grained reactivity. No VDOM. No diffing. Just signals.
